# crypto-pipeline

Market data ETL and streaming pipeline for cryptocurrency prices. Pulls historical OHLCV candle data, streams real-time price ticks, computes financial metrics (SMA, volatility, VWAP), and stores everything in local SQLite.

> **This is a proof-of-concept build** generated autonomously by [Software Factory](https://github.com/aldousbirchall/softwarefactory) to validate the factory's data pipeline capabilities. Crypto was chosen because markets run 24/7, making it a convenient test harness for any time of day. The provider abstraction is reusable for equities, FX, or commodities feeds. Not production software.

## Install

```bash
pip install -e .
```

Requires Python 3.9+. External dependencies: `requests`, `websocket-client`.

## Usage

```bash
# Pull 30 days of hourly candle data
crypto-pipeline backfill --assets bitcoin,ethereum

# Incremental update (only new data since last pull)
crypto-pipeline refresh --assets bitcoin,ethereum

# Real-time price streaming via WebSocket
crypto-pipeline stream --assets bitcoin,ethereum

# Query latest prices
crypto-pipeline query latest --assets bitcoin

# Query historical data with computed metrics
crypto-pipeline query history --start 2026-01-01 --end 2026-02-01 --assets bitcoin
```

### Global Flags

| Flag | Default | Description |
|---|---|---|
| `--assets` | `bitcoin,ethereum` | Comma-separated asset IDs |
| `--db` | `crypto_pipeline.db` | SQLite database path |
| `--provider` | `binance` | Data provider (`binance`, `coingecko`, `coincap`) |

### Output Format

- `query latest`: tab-separated `ASSET  PRICE  TIMESTAMP`
- `query history`: tab-separated with header, includes SMA-20, SMA-50, volatility, VWAP columns

## Providers

Pluggable provider architecture via `CryptoProvider` abstract base class:

| Provider | OHLCV | WebSocket | Notes |
|---|---|---|---|
| Binance | Hourly, with volume | Yes | Default. Full-featured. |
| CoinGecko | 4-hour, no volume | No (polling) | Free tier, no API key needed |
| CoinCap | Hourly | Yes | API currently down (since Feb 2026) |

## Computed Metrics

| Metric | Method |
|---|---|
| SMA-20/50 | Simple moving average of closing prices |
| Volatility | Rolling 20-period standard deviation of log returns |
| VWAP | Volume-weighted average price with daily reset |

Metrics are null where insufficient history exists.

## Architecture

```
cli.py          argparse subcommand dispatch
pipeline.py     orchestration (backfill, refresh, stream, query)
api_client.py   provider implementations (HTTP + WebSocket)
metrics.py      pure functions, no I/O
storage.py      SQLite via stdlib
validation.py   NaN/Inf filtering
```
